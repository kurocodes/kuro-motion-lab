# ⚡ Kuro Motion Lab

A collection of UI components… <br/>
but not the static kind.

Small interactions. <br/>
Motion-driven behavior. <br/>
Things that respond, not just render.

This isn’t a component library— <br/>
it’s a place to explore interaction.

## 🧪 What is this?

A modern component showcase where:

* each component is **interactive**
* each interaction is **intentional**
* each idea is **explored, not just displayed**

Click a component → open its page → try it.

No screenshots. <br/>
No imagination required.

## 🧠 Why this exists

Most component libraries show:

> what it looks like

This focuses on:

> how it behaves

Because good UI isn’t just layout… <br/>
it’s motion, feedback, and timing.

A growing set of interaction-driven components:

* 🧬 Morphing menus

## ✨ Core Features

* ⚡ Live interactive previews (not static demos)
* 🧠 Data-driven component system
* 🌓 Light / dark theme support
* 📦 Lazy-loaded components (optimized bundle)
* 📱 Responsive layout with graceful fallbacks
* 🖥️ Device-aware UX (hover / cursor-based components handled properly)
* 🔗 Optional GitHub links for each component

## 🛠️ Tech Stack

* React + Vite + TypeScript
* Tailwind CSS
* Motion (animations)
* React Router
* Lucide Icons

Simple tools.

Focused execution.

## 🧱 Architecture

Built with clarity in mind:

* Components are **lazy-loaded per route**
* Data drives rendering (`componentsData`)
* UI layout is separated from component logic
* Demo data is isolated from core systems
* Each component remains **independent and reusable**

No tight coupling. <br/>
No unnecessary complexity.

## 🧠 Interaction Principles

Every component follows a few rules:

* motion should feel continuous
* interactions should feel responsive
* visuals should support behavior—not distract from it

Small details matter:

* hover timing
* easing curves
* spacing
* feedback

That’s where good UI becomes great.

## 📱 Device Handling

Some interactions rely on:

* hover
* cursor tracking

On touch devices:

* components remain visible
* unsupported interactions are gracefully disabled
* clear messaging is shown:

> “This interaction uses hover or cursor tracking.
> Try it on desktop for full experience.”

No broken experiences. <br/>
No hidden content.
